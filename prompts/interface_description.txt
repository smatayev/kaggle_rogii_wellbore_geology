You are a senior full-stack engineer. Build a complete local web application from scratch for visualizing horizontal well trajectories and geological cross-sections using CSV files stored in a local data/ folder.

Your goal is to generate a clean, modular, maintainable implementation that is easy to run locally and easy to extend later.

## High-level objective

Create an engineering-style dashboard application that allows the user to:

1. Select a horizontal well from a dropdown
2. Select a typewell from a dropdown
3. View multiple synchronized plots showing:
   - Gamma Ray along the horizontal well
   - Well path projection on a vertical plane
   - TVT-based comparison between horizontal well and typewell

The UI should resemble a technical interpretation dashboard, similar to a geology/well placement visualization tool.

## Dataset description

The data represents horizontal well trajectories and vertical reference logs used for geological prediction.

Each well is identified by a unique 8-character hash, for example:
015fe0d2

### Horizontal well file columns

Required columns:
- MD: Measured Depth (ft)
- X: Easting (ft)
- Y: Northing (ft)
- Z: True Vertical Depth (ft)
- GR: Gamma Ray (API)

Optional / expected columns:
- ANCC
- ASTNU
- ASTNL
- EGFDU
- EGFDL
- BUDA
- TVT
- TVT_input

Meaning:
- Formation columns represent predicted depth of geological formations
- TVT is the interpreted geological position along the lateral
- TVT_input is a copy of TVT, but contains NaN values in the evaluation/prediction zone

### Typewell file columns

Required columns:
- TVT
- GR
- Geology

Meaning:
- TVT is the vertical depth index
- GR is the vertical gamma ray signature
- Geology is a categorical formation label such as EGFDU, EGFDL, BUDA, etc.

## Chart details

### 1) Gamma Ray Log

Use the selected horizontal well.

- Plot GR vs MD
- Black line for GR
- Green filled area under the line
- Title: Gamma Ray Log
- X-axis label: MD (FT)
- Y-axis label: GR

If GR or MD is missing, show a clear warning message instead of crashing.

---

### 2) Well Path Projection on Vertical Plane

Use the selected horizontal well.

This chart should show:
- Projected well path
- Geological surface lines if available
- Prediction start marker if derivable

#### Required logic

Compute azimuth:
- Use the first valid (X, Y) point and the last valid (X, Y) point
- Compute azimuth in degrees

Compute projected horizontal distance:
- Project the trajectory onto a vertical plane aligned with the azimuth
- X-axis = cumulative projected distance along azimuth direction
- Y-axis = Z depth
- Depth should visually increase downward

Plot:
- Projected well path as thick blue line
- If TVT_input contains NaN values:
  - find the first row where TVT_input is NaN
  - plot a large red marker at that location
  - label: Prediction start (MD=<value> ft)

If these formation columns exist, overlay them as separate colored lines:
- ANCC
- ASTNU
- ASTNL
- EGFDU
- EGFDL
- BUDA

Use consistent colors such as:
- ANCC = red
- ASTNU = orange
- ASTNL = purple
- EGFDU = green
- EGFDL = cyan
- BUDA = magenta

Title format:
Well Path Projection on Vertical Plane (Azimuth = <value>°)

Axis labels:
- X-axis: Horizontal Distance Along Azimuth
- Y-axis: Depth (Z, FT)

---

### 3) TVT plot

This chart compares horizontal-well GR and typewell GR in TVT space.

Requirements:
- Plot horizontal well GR vs horizontal well TVT in black
- Plot typewell GR vs typewell TVT in red
- X-axis label: GR
- Y-axis label: TVT (FT)
- Depth should increase downward
- Set GR display range to 0–200 by default if practical
- Show a small annotation or label near the top: GR range: 0 - 200

Geology boundary markers:
- Derive boundary depths from changes in the Typewell Geology column
- Whenever the Geology value changes between adjacent rows, record the TVT
- Draw horizontal dashed lines at those TVT values
- If the formation name matches one of:
  ANCC, ASTNU, ASTNL, EGFDU, EGFDL, BUDA
  use the same color as in the well-path chart
- Otherwise use a neutral dashed gray line

Title:
TVT plot

Important:
- If the selected horizontal well does not have TVT, display a warning and skip this chart gracefully
- If the typewell lacks Geology, render the plot without boundary markers

---

### 4) TVT plot (last 200 FT)

This chart is the same as TVT plot, but zoomed into the last 200 ft of the TVT range.

Requirements:
- Same two overlaid curves
- Same boundary markers
- Default y-range should be:
  [max TVT - 200, max TVT]
- Title:
  TVT plot (last 200 FT)

If max TVT is unavailable, show a meaningful warning.

- TVT plot unavailable because horizontal well has no TVT column

## Implementation details and assumptions

Use these assumptions unless better justified in code comments:

1. Gamma Ray chart x-axis should use MD.
2. Azimuth can be computed from first valid and last valid trajectory points.
3. Projected horizontal distance can be derived by projecting XY coordinates onto the azimuth unit vector.
4. For the well path chart, use Z depth values directly on the y-axis.
5. Prediction start is the first row where TVT_input becomes NaN.
6. Geological boundaries in TVT plots come from transitions in typewell Geology.
7. The “last 200 FT” TVT chart should default to the deepest 200 ft interval.
8. If any optional formation columns are missing, still render the chart with available data.
9. If a selected file is invalid, show a warning in the UI instead of raising an unhandled exception.
